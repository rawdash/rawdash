Rawdash exposes a [Model Context Protocol](https://modelcontextprotocol.io/)
(MCP) server so Claude-powered tools can query your dashboard data as context.

## Prerequisites

- A Rawdash config and a storage backend (see [OSS Quickstart](/docs/quickstart))
- [Claude Code](https://claude.ai/code) or [Claude Desktop](https://claude.ai/download)

## How it works

`@rawdash/mcp` is a library, not a standalone CLI. It exports
`createMcpServer({ config, storage })`, which returns an MCP server bound to your
Rawdash config and storage. You write a small entry script that connects it to a
stdio transport, then point Claude Code or Claude Desktop at that script with
`node`. The server reads directly from your storage — there's no HTTP URL to
configure.

## Write the entry script

<Steps>

1. **Install the package**

   ```sh
   npm install @rawdash/mcp @modelcontextprotocol/sdk
   ```

2. **Create the server entry point**

   Reuse the same `config` and `storage` you built in the Quickstart:

   ```ts title="mcp-server.ts"
   import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio';
   import { createMcpServer } from '@rawdash/mcp';

   import { config, storage } from './rawdash.config';

   const server = createMcpServer({ config, storage });
   await server.connect(new StdioServerTransport());
   ```

   Export `config` (from `defineConfig`) and `storage` (e.g. a `LibsqlStorage`
   instance) from your config module so the script above can import them.

3. **Build it to a runnable JS file**

   Compile `mcp-server.ts` with your bundler/`tsc` (or run it through `tsx`).
   Note the absolute path to the resulting entry point — you'll reference it from
   the Claude config below.

</Steps>

## Claude Code

<Steps>

1. **Open your project settings**

   In your project root, open (or create) `.claude/settings.json`.

2. **Add the Rawdash MCP server**

   ```json title=".claude/settings.json"
   {
     "mcpServers": {
       "rawdash": {
         "command": "node",
         "args": ["/absolute/path/to/mcp-server.js"]
       }
     }
   }
   ```

   Point `args` at the entry script you built above. Add any env vars your
   `rawdash.config` needs (e.g. `GITHUB_TOKEN`, `TURSO_URL`) under an `"env"` key.

3. **Restart Claude Code**

   Claude Code picks up MCP server changes on restart. After restarting, type
   `/mcp` to verify the `rawdash` server is listed.

4. **Ask questions about your data**

   Once connected, Claude can query your dashboards directly:

   > "What were the most active repos last week?"
   >
   > "Show me open PRs older than 7 days across all repos."

</Steps>

<Steps>

1. **Open the Claude Desktop config file**

   <Tabs>

     </TabItem>
     <TabItem label="Windows">

     </TabItem>
   </Tabs>

2. **Add the Rawdash MCP server**

   ```json
   {
     "mcpServers": {
       "rawdash": {
         "command": "node",
         "args": ["/absolute/path/to/mcp-server.js"]
       }
     }
   }
   ```

   Use the same entry script described in [Write the entry
   script](#write-the-entry-script). Add any env vars your config needs under an
   `"env"` key.

3. **Restart Claude Desktop**

   The Rawdash tools will appear in the Claude Desktop tools panel after restart.

</Steps>

## Available MCP tools

Once connected, Claude has access to these tools:

| Tool | Description |
|------|-------------|
| `list_dashboards` | List all dashboards available on this Rawdash instance |
| `list_widgets` | List all widgets on a dashboard |
| `read_widget` | Fetch resolved widget data (same shape as the HTTP read API) |
| `render_widget` | Render a widget as a human-readable markdown summary |
| `list_connectors` | List all configured connectors and their sync status |
| `add_connector` | Add a connector instance (OSS: runtime-only, not persisted) |
| `remove_connector` | Remove a connector instance (OSS: runtime-only) |
| `trigger_sync` | Kick off a sync for one or all connectors |
| `list_secrets` | List secret names known to this instance (values never returned) |
| `set_secret` | Set a secret for the running instance (OSS: stored in `process.env`) |

## Scoping access

The MCP server exposes every dashboard in the `config` you pass to
`createMcpServer`. To restrict what Claude can see, build a narrower
`DashboardConfig` (with only the dashboards you want to expose) in your entry
script and pass that instead of your full config.
