This guide walks you through installing Rawdash, configuring a GitHub connector,
and mounting the request handler in a Node.js server.

<Steps>

1. **Install packages**

   <Tabs>
     <TabItem label="npm">
       ```sh
       npm install @rawdash/core @rawdash/server @rawdash/hono @rawdash/connector-github @rawdash/adapter-turso @hono/node-server
       ```
     </TabItem>
     <TabItem label="pnpm">
       ```sh
       pnpm add @rawdash/core @rawdash/server @rawdash/hono @rawdash/connector-github @rawdash/adapter-turso @hono/node-server
       ```
     </TabItem>
     <TabItem label="yarn">
       ```sh
       yarn add @rawdash/core @rawdash/server @rawdash/hono @rawdash/connector-github @rawdash/adapter-turso @hono/node-server
       ```
     </TabItem>
   </Tabs>

   If you are mounting Rawdash inside a Next.js app, also install `@rawdash/sdk-nextjs`:

   <Tabs>
     <TabItem label="npm">
       ```sh
       npm install @rawdash/sdk-nextjs
       ```
     </TabItem>
     <TabItem label="pnpm">
       ```sh
       pnpm add @rawdash/sdk-nextjs
       ```
     </TabItem>
     <TabItem label="yarn">
       ```sh
       yarn add @rawdash/sdk-nextjs
       ```
     </TabItem>
   </Tabs>

2. **Create your config file**

   ```ts title="rawdash.config.ts"
   import { serve as honoServe } from '@hono/node-server';
   import { defineConfig, defineDashboard, secret } from '@rawdash/core';
   import { GitHubActionsConnector } from '@rawdash/connector-github';
   import { TursoStorage } from '@rawdash/adapter-turso';
   import { mountEngine } from '@rawdash/hono';

   const { app } = mountEngine(
     defineConfig({
       connectors: [
         {
           name: 'main',
           connectorId: 'github-actions',
           config: {
             owner: 'myorg',
             repo: 'backend',
             token: secret('GITHUB_TOKEN'),
           },
         },
       ],
       dashboards: {
         eng: defineDashboard({ widgets: {} }),
       },
     }),
     {
       connectorRegistry: { 'github-actions': GitHubActionsConnector },
       storage: new TursoStorage({
         url: process.env.TURSO_URL ?? 'file:rawdash.db',
         authToken: process.env.TURSO_AUTH_TOKEN,
       }),
     },
   );
   honoServe({ fetch: app.fetch, port: 8080 });
   ```

3. **Connect your frontend**

   The config file from Step 2 starts a standalone HTTP server when run with
   `tsx`. From your frontend (React, Vue, vanilla JS, etc.) use the
   framework-agnostic `@rawdash/sdk-client` to query it — or
   `@rawdash/sdk-nextjs` if you're inside a Next.js App Router app (see Step 1
   above for the install commands):

   ```ts title="lib/rawdash.ts"
   import { http } from '@rawdash/sdk-client';

   export const rawdash = http({
     baseUrl: process.env.RAWDASH_URL ?? 'http://localhost:8080',
   });
   ```

4. **Set environment variables**

   ```sh title=".env.local"
   GITHUB_TOKEN=ghp_...
   TURSO_URL=libsql://your-db.turso.io
   TURSO_AUTH_TOKEN=ey...
   ```

   <Aside type="tip">
     For local development without Turso, use an in-memory SQLite URL:
     `TURSO_URL=file::memory:?cache=shared`
   </Aside>

5. **Start the server and verify**

   ```sh
   npx tsx rawdash.config.ts
   ```

   Rawdash will start on port 8080, run the initial sync, then enter the
   background polling loop. Check liveness, then sync state:

   ```sh
   curl http://localhost:8080/health
   # {"status":"ok"}
   curl http://localhost:8080/sync/state
   # {"status":"succeeded","queuedAt":null,"startedAt":null,"lastSyncAt":"2026-04-25T10:00:00.000Z","lastError":null}
   ```

6. **Query from your frontend**

   ```ts
   const res = await fetch('http://localhost:8080/dashboards/eng/widgets');
   const widgets = await res.json(); // CachedWidget[]
   ```

</Steps>
